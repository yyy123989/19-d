# AD9910 全功能菜单移植设计

日期：2026-05-22

## 目标

在当前 `TFT+AD7060B` 工程中，把参考资料里的 AD9910 功能完整移植进现有 STM32F407VET6 + TFT + AD7606B 系统。移植后系统应保留现有 TFT 显示、AD7606B 定时采样、AD9910 输出能力，并通过板载两个按键完成模式选择和参数调整。

本轮只做设计确认，不烧录。后续实现完成后只做代码检查和编译验证，除非用户明确要求烧录。

## 当前硬件边界

AD9910 继续使用当前接线，不额外要求改线：

| AD9910 信号 | STM32F407VET6 | 用途 |
| --- | --- | --- |
| CS / CSB | PB10 | 软件 SPI 片选 |
| SCLK | PB13 | 软件 SPI 时钟 |
| SDIO / MOSI | PB15 | 软件 SPI 数据 |
| IO_UPDATE | PB12 | AD9910 参数更新 |
| MASTER_RESET | PA8 | AD9910 复位 |
| PWR_DWN | PB9 | 电源控制 |
| PROFILE0 | PE2 | 当前只保留单 profile 控制能力 |
| PROFILE1 | GND | 按用户要求接地 |
| PROFILE2 | GND | 按用户要求接地 |
| OSK | PE5 | 幅度键控控制 |
| DRCTL | PE6 | DRG 扫频方向控制 |
| DRHOLD | PB8 | DRG 保持控制 |
| DDS OUT | 示波器 / AD7606B CHx | 输出观测与采集 |

板载按键使用：

| 按键 | STM32F407VET6 | 说明 |
| --- | --- | --- |
| KEY1 | PE3 | 菜单移动 / 返回 |
| KEY0 | PE4 | 参数切换 / 确认 |

TFT 使用商家预留 FSMC 接口，不改变 TFT 相关引脚。AD7606B 保持当前串行采样方案和 TIM3 固定触发采样方案。

注意：参考资料和部分模块说明里 AD9910 软件 SPI 曾写成 SCLK=PB3、SDIO=PB5；本工程实际接线和 CubeMX 定义是 SCLK=PB13、SDIO=PB15。后续代码注释必须明确“以当前工程接线为准”，避免再按参考资料误改。

## 移植范围

### 1. 单频模式

单频模式支持选择频率、幅度、相位，然后输出正弦波。参数表沿用参考资料中的常用档位：

- 频率：`0, 1, 10, 100, 500, 1k, 5k, 10k, 50k, 100k, 500k, 1M, 5M, 10M, 50M, 100M, 200M, 300M, 380M, 400M, 420M`
- 幅度：`0, 1023, 2047, 4095, 8191, 9215, 10239, 11263, 12287, 13311, 14335, 15359, 16383`
- 相位：`0, 90, 180, 270, 360`

参数改变后不会立即频繁写寄存器，用户确认输出时再统一写 AD9910，避免菜单操作时输出乱跳。

### 2. RAM 波形播放模式

RAM 模式移植参考资料中的 6 个子模式：

1. 直接转换模式
2. 上斜坡模式
3. 双向斜坡模式
4. 连续双向斜坡模式
5. 连续循环模式
6. 三角波和方波

实现上优先使用 AD9910 串行寄存器和 RAM 写入，不依赖 PROFILE1 / PROFILE2，因为当前硬件把它们接地。每个子模式提供固定可运行参数，后续可以再扩展成可编辑参数。

RAM 移植时不复刻参考资料中通过 `Set_Profile(0~4)` 轮流切换 Profile 的演示逻辑。当前硬件只有 PROFILE0 可控，PROFILE1 / PROFILE2 已接地，所以所有 RAM 子模式都只写和回放 Profile 0；需要切换 RAM 目标或波形时，重新写 Profile 0 对应寄存器和 RAM 数据。

RAM 数据表需要从完整 AD9910 V1.2 参考源码中移植，至少包含 `RAM_AMP[]`、`RAM_Fre[]`、`RAM_PHA[]` 三个 1024 点 32 位表。三角波 / 方波子模式如使用参考资料中的专用表，也一并放入独立 RAM 表文件，避免继续用临时生成表替代原资料。

### 3. DRG 扫频模式

DRG 模式输出频率线性扫频。当前 1kHz 到 10kHz 扫频已经跑通，本次移植保留并菜单化：

- 起始频率
- 终止频率
- 步进频率
- 扫频速度档位
- 上扫 / 下扫 / 往返扫

方向翻转继续由 TIM4 处理，避免被 AD7606B 高速采样和 TFT 刷新拖慢。TIM3 继续负责 AD7606B 采样。

### 4. OSK 幅度键控模式

OSK 模式使用 PE5 控制 AD9910 的 OSK 引脚。菜单中提供：

- OSK 关闭：保持普通连续输出
- OSK 手动开关：KEY0 控制幅度开 / 关
- OSK 自动闪烁：用软件定时周期性切换 OSK

第一版不新增外部触发线，先让板载按键可以验证幅度键控是否生效。

### 5. 参数校准模式

参考资料中的参数校准包含偏移量和增益调节，但原工程有并口调制相关逻辑，需要额外并行数据线。当前 AD9910 只接了串行控制线，无法原样复刻并口校准。

本工程先实现软件校准层：

- 频率偏移：在用户选择频率后叠加 `offset_fre` 档位，再写入 FTW
- 幅度增益：在用户选择幅度后按 `gain` 档位缩放，再写入 ASF。软件映射先采用 `gain=8` 为 1.00 倍，`gain=0` 到 `15` 对应约 0.50 倍到 1.44 倍，最终 ASF 裁剪到 16383。
- 相位偏移：在用户选择相位后叠加软件相位偏移，再写入 POW

可选档位沿用参考资料：

- 频率偏移：`0, 1k, 2k, 4k, 8k, 10k, 20k, 40k, 60k`
- 增益档位：`0` 到 `15`，默认 `8`

如果后续需要完全复刻原参考资料的并口校准，需要重新增加 AD9910 并口相关接线和引脚规划。

## 两键菜单设计

由于只有 KEY1 / KEY0 两个板载按键，使用短按和长按组合完成菜单操作：

长按阈值统一定义为 500 ms，按键去抖仍使用 30 ms。

| 操作 | 行为 |
| --- | --- |
| KEY1 短按 | 移动到下一个菜单项或下一个参数 |
| KEY0 短按 | 修改当前参数值或切换当前选项 |
| KEY1 长按 | 返回上一级页面 |
| KEY0 长按 | 进入子菜单或应用当前输出 |

主菜单页面：

- `M1 SINGLE`
- `M2 RAM`
- `M3 DRG`
- `M4 OSK`
- `M5 CAL`
- `M6 AD VIEW`

TFT 所有 DDS 页面都保留混合顶栏：显示当前 DDS 模式 / 关键参数，同时显示 AD7606B 实时状态、SPS、错误计数和至少一路实时通道值（默认 CH1）。`AD VIEW` 仍作为独立页面集中显示 CH1~CH8 全部实时值，这样调 DDS 时也不会完全看不到 ADC 反馈。

## 软件结构

### `ad9910.c / ad9910.h`

保留底层 bit-bang SPI 和已有初始化逻辑，扩展为完整 AD9910 驱动接口：

- 单频输出：频率 / 幅度 / 相位
- DRG 频率扫频：上下限 / 步进 / 速率 / 方向
- RAM 写入与 6 种 RAM 播放配置
- OSK 使能和 OSK 引脚控制
- 参数边界裁剪与寄存器值计算

驱动层只负责把参数写进 AD9910，不处理菜单状态。

### `ad9910_ram_tables.c / ad9910_ram_tables.h`

新增独立 RAM 数据表文件，从 `D:\Users\港城泽人\Desktop\模块\ad9910\AD9910模块资料\测试程序\AD9910 V1.2\User\AD9910.c` 搬运 `RAM_AMP[]`、`RAM_Fre[]`、`RAM_PHA[]`。表以 `const uint32_t` 保存，供 `ad9910.c` 写入 AD9910 RAM 使用。

### `dds_app.c / dds_app.h`

新增 DDS 应用层，负责保存当前模式和参数：

- 当前模式
- 单频参数索引
- RAM 子模式
- DRG 参数索引
- OSK 状态
- 校准参数

应用层提供 `DDS_AppApply()` 一类接口，由菜单确认时调用。这样主循环不直接散落 AD9910 写寄存器逻辑。

### `ui_menu.c / ui_menu.h`

新增两键菜单状态机：

- 按键去抖
- 短按 / 长按识别
- 页面切换
- 当前光标位置
- 参数显示文本生成

菜单层不直接写 AD9910，只调用 `dds_app` 的参数修改和应用接口。

### `main.c`

`main.c` 只保留系统初始化、TIM3/TIM4 启动、主循环调度：

- AD7606B 采样继续由 TIM3 触发
- DRG 方向切换继续由 TIM4 触发
- TFT 刷新和菜单扫描在主循环中运行
- AD9910 具体功能下沉到 `dds_app` 和 `ad9910`

## 数据流

1. 用户按键进入菜单并修改参数。
2. `ui_menu` 解析按键事件，更新菜单光标或参数索引。
3. 用户长按 KEY0 应用输出。
4. `dds_app` 根据当前模式生成 AD9910 参数。
5. `ad9910` 写寄存器并执行 `IO_UPDATE`。
6. TFT 刷新当前模式、参数和 AD7606B 状态。

AD7606B 采样链路与 DDS 菜单链路分开，避免菜单刷新影响采样触发。

## 边界处理

- 频率最大限制为 420MHz，超过后自动裁剪。
- 幅度最大限制为 16383，增益校准后仍不能超过该值。
- 相位按 360 度取模。
- DRG 要求终止频率大于起始频率，步进和速率不能为 0。
- RAM 写入前检查 RAM 数据长度，避免超过 AD9910 RAM 容量。
- 当前硬件没有 PROFILE1 / PROFILE2 控制线，代码中不依赖它们切换功能。
- 当前硬件没有 AD9910 并口调制线，校准模式明确采用软件校准。

## 验证策略

实现阶段每一步完成后只做检查和编译，不烧录：

- 静态检查：确认 KEY0/KEY1 没有和 PROFILE 引脚冲突。
- 静态检查：确认没有重新引入 PROFILE1 / PROFILE2 软件控制。
- 静态检查：确认长按阈值只有一个 `500U` 定义。
- 静态检查：确认 AD9910 SCLK/SDIO 代码注释说明 PB13/PB15 是当前工程实际接线。
- 静态检查：确认 `RAM_AMP[]`、`RAM_Fre[]`、`RAM_PHA[]` 已移植到工程内，RAM 子模式没有调用 `Set_Profile(1~4)`。
- 静态检查：确认 DRG 方向仍由 TIM4 服务，不依赖主循环延时。
- 编译检查：使用当前 Keil ARMCC 工具链分别编译改动过的 `.c` 文件。
- 行为检查：根据代码路径确认单频、RAM、DRG、OSK、校准都会走到 AD9910 写寄存器和 `IO_UPDATE`。

硬件现象验证由用户烧录后观察示波器和 TFT，再根据现象继续调参数。
